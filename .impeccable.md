## Design Context

### Users
Claude Workflow Composer is for developers and technically curious builders who use Claude Code repeatedly and want to turn recurring work into reusable local workflows. They need to detect patterns in their own history, compose existing agents and skills, export reliable workflows, run them headlessly, and monitor approvals without losing trust in what the tool is doing.

### Brand Personality
Precise, inevitable, calm. The interface should feel like a serious local automation console: quiet enough for long sessions, clear enough for non-experts, and credible enough that power users trust it with real coding workflows.

### Aesthetic Direction
Use refined utilitarian product design rather than decorative SaaS styling. Keep surfaces mostly neutral, reserve accent color for focus, primary action, active states, and live workflow paths, and make the graph/canvas and evidence-backed Detect flow the memorable parts. Avoid AI slop: no purple/cyan gradients, ornamental glow, repeated promo cards, centered marketing composition, emoji-driven status systems, or decorative grids outside the canvas.

### Design Principles
- Make the product feel inevitable: every control should look like it belongs exactly where it is.
- Prefer panes, rows, rails, and compact lists over card-heavy layouts.
- Treat Detect as a product surface, not a debug page: evidence and promotion should lead; logs should support.
- Use visual emphasis sparingly so important workflow state is unmistakable.
- Preserve speed and clarity over flourish; motion should explain state changes only.
